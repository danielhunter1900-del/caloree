Generated from pain history + linked workouts · tap Ask Claude on Trends for deeper analysis.
Diagonal stripes overlay days where pain ≥ 3 was logged (thicker = higher). Rest days keep their 🌙. Pattern card at bottom correlates pain dates to prior-day training load. Filter chip Pain overlay is togglable so the grid stays clean by default.

Data model: new table pain_logs: {id, user_id, date, body_part, severity, type[], aggravators[], note, linked_workout_id, resolved_at}. One entry per part per day (upsert). Body map is ~20 named SVG regions. Severity scale is deliberately short (0–5) to stay loggable in 2 taps. Pain levels ≥ 3 push a stripe onto Trends day-cells; levels 4–5 push a thicker stripe. Claude's pattern analysis reuses the existing Ask-Coach flow, now with pain_logs as additional context.
