Home page itself splits into two modes — Train or Track — via a toggle in the header. 4-tab layout preserved. The mode defaults based on time: morning = Train, evening = Track.

Pros

No extra tab. Both views share the day header, so the context (which day) is always consistent.

Cons

One view is always hidden — need to tap to switch. Could feel modal if you use both equally every day.

Plan sits as a banner at the top of Home. The existing "Home sections" pill adds a Plan segment alongside Macros / Wellness / Exercise so the full plan view expands in place.

Pros

Zero new navigation. Plan peeks in the banner every time you open the app. One place for everything today.

Cons

Home gets long. Multi-week planner view needs a separate screen anyway. Banner always there even on rest days.

Recommendation: A (Plan tab) is the cleanest and most extensible. It keeps the existing Home/Log/Trends/Profile mental model intact and gives the plan its own real estate — important when the multi-week view, preferences, equipment, adaptive recompute, and history all live inside it. C (banner + segment) is nice as a complement: ship A, and add the compact banner on Home so you see today's plan immediately even without switching tabs. B feels too modal given both views are frequent.
