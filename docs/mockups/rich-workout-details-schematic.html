Implementation plan: the existing ExerciseDetailSheet already branches on entry.type. Add three new type-specific sections gated on data presence:
• Calisthenics: show sets table when details.sets exists (already collected at logging time). Muscle coverage reuses the EXERCISE_MUSCLE_MAP. PR detection is client-side — scan same-movement history for max reps / weight / hold.
• Swim: Strava sync exposes lengths, strokes, SWOLF when the device provides them. Add a lane chart with per-length pace, stroke breakdown, and SWOLF delta card.
• Cycling: power / cadence / NP / elevation sparkline, all already in details.*. Just needs rendering. Map + vs-last-4 are shared with runs.
