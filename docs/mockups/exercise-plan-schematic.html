How it went (optional)
Session broken into checkable blocks. Strava imports auto-tick matching blocks (if type/duration/distance fits). RPE + note feed Claude analysis. "Rest instead" cancels the session and flips the day to rest without losing the plan cadence.

Data model: training_plans (id, user_id, template, start_date, week_count) + plan_sessions (plan_id, date, kind ['run'|'cal'|'plyo'|'mob'|'rest'], blocks[jsonb], completed_at, rpe, note, matched_exercise_id). Plan generation is deterministic from the chosen template — Claude isn't in the loop for generation. Completion inference: when a new exercise is logged (manual or Strava), match to the nearest scheduled plan_session (same date, same kind, ±30% duration); auto-complete if matched. Claude can analyse adherence weekly: "You skipped 2 of 3 plyo sessions this month — want me to adjust?"
